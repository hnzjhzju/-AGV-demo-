## 一、系统架构概述

本系统采用**三层架构**设计，实现从任务调度到车端执行的全链路管理：

```
┌─────────────────────────────────────────────┐
│              调度层（Dispatcher）              │
│  任务队列管理 · 负载均衡分配 · 全局优化         │
└────────────────────┬────────────────────────┘
                     │ 任务指令下发
┌────────────────────▼────────────────────────┐
│              边缘层（Edge）                   │
│  冲突检测 · 资源锁 · 死锁检测与解除            │
└────────────────────┬────────────────────────┘
                     │ 路径 + 执行指令
┌────────────────────▼────────────────────────┐
│              车端模型（AGV）                  │
│  状态机 · 路径执行 · 电量管理 · 位置上报       │
└─────────────────────────────────────────────┘
```

## 二、核心算法设计

### 2.1 A* 路径规划（加权变种）

- **启发函数**：曼哈顿距离（网格地图最优）
- **权重系数**：`weight = 1.2`（速度优先，次优解）
- **转向惩罚**：`+0.3` cost，减少路径折返
- **路径平滑**：去除共线冗余节点（减少不必要的停顿点）

```
f(n) = g(n) + w × h(n)
其中：h(n) = |Δx| + |Δy|（曼哈顿距离）
```

### 2.2 任务分配策略

**贪心最近 AGV 优先**（可扩展为匈牙利算法最优分配）：

1. 按优先级排序待分配任务（紧急 > 普通）
2. 对每个任务，从空闲 AGV 中选择曼哈顿距离最近者
3. 电量低于 20% 的 AGV 不参与分配

### 2.3 冲突避免（资源锁机制）

```
路径预占（Path Reservation）：
  AGV 规划路径后，锁定路径上的所有节点
  其他 AGV 规划路径时，被占用节点视为障碍
  AGV 离开节点后，释放该节点的占用
```

### 2.4 死锁检测与解除

**超时检测法**（简化实现）：
- `blocked_ticks >= 8` → 判定死锁
- 解除策略：释放当前路径预占 → 等待随机 tick → A* 重规划

## 三、港口场景设计

| 区域 | 数量 | 说明 |
|------|------|------|
| 集装箱堆场（障碍）| 9 块 | 分布于 3 列，构成主要绕行障碍 |
| 充电桩 | 3 个 | 左侧纵向排列 |
| 装货点（Pickup）| 9 个 | 堆场左侧通道 |
| 卸货点（Dropoff）| 7 个 | 右侧港口出口区 |
| AGV 车队 | 15 台 | 初始停于充电桩 |

**地图尺寸**：40×25 网格（800×500 逻辑像素）
